## The 7 Stages

### Stage 1: Discovery

**Goal:** Capture attention in 3 seconds.

```
WHERE:
├── Twitter/X (Crypto Twitter)
├── Discord servers
├── Telegram groups
├── Reddit (r/solana, r/cryptocurrency)
└── Word of mouth

HOOKS THAT WORK:
├── Fear: "Is your token about to rug?"
├── Greed: "Find healthy tokens before they moon"
├── Social proof: "K-Score 85 - verified by 10K users"
└── Curiosity: "Why do whales check this before buying?"

FORMAT:
├── K-Score cards (shareable PNG)
├── Short-form video (< 60 seconds)
├── Thread format (hook + value + CTA)
└── Memes (organic spread)
```

**The Hook:** "K-Score: Don't trust tokens, verify them."

**Emotion:** Fear of rugs → Promise of safety.

---

### Stage 2: First Value

**Goal:** Deliver value in under 30 seconds, no friction.

```
REQUIREMENTS:
├── No signup required
├── No wallet connection required
├── No payment required
├── Instant result
└── Mobile-friendly

USER ACTION:
├── Arrive at HolDex
├── Enter token address (or search)
├── See K-Score immediately
└── Understand result (color-coded)

FIRST VALUE:
├── "My token is healthy" → Confidence
├── "My token is risky" → Warning (saved them!)
├── "This matches my intuition" → Trust established
└── "I learned something new" → Curiosity sparked
```

**Metric:** Time to first value < 30 seconds.

**Critical:** The first experience MUST be positive, even if the K-Score is low. A warning is still value.

---

### Stage 3: Exploration

**Goal:** User checks multiple tokens, learns the system.

```
BEHAVIORS:
├── Check their portfolio tokens
├── Check tokens from CT discussions
├── Compare high vs low K-Scores
├── Read D/O/L explanations
└── Click around the interface

LEARNING MOMENTS:
├── "D means Diamond Hands (conviction)"
├── "O means Organic Growth (distribution)"
├── "L means Longevity (survival)"
├── "Geometric mean = all three matter"
└── "100 = SOL, 90+ = exceptional"

AHA MOMENT:
"I found a hidden gem with K=85 that CT hasn't discovered yet."
```

**Metric:** Tokens checked per session > 3.

**Critical:** Education must be progressive, not overwhelming. Show basics first, details on demand.

---

### Stage 4: Investment

**Goal:** User invests time/attention into the ecosystem.

```
INVESTMENT TYPES:
├── Create watchlist (track tokens)
├── Import portfolio (wallet scan)
├── Set alerts (K-Score thresholds)
├── Make predictions (stake on outcomes)
└── Share cards (social investment)

WHY INVESTMENT MATTERS:
├── Sunk cost → return likelihood ↑
├── Personalization → relevance ↑
├── Stake → engagement ↑
└── Identity → loyalty ↑
```

**Current Gap:** This stage is largely missing. Users check K-Score and leave.

**Required Features:**
- Watchlist with saved tokens
- Portfolio mode (connect wallet, see all holdings)
- Alert system (notify when K-Score changes)
- Prediction tracking (prove K-Score works)

---

### Stage 5: Habit

**Goal:** User returns daily without external prompt.

```
THE HOOKED MODEL:

┌──────────┐    ┌──────────┐    ┌──────────────┐    ┌────────────┐
│ TRIGGER  │───►│  ACTION  │───►│   VARIABLE   │───►│ INVESTMENT │
│          │    │          │    │    REWARD    │    │            │
└──────────┘    └──────────┘    └──────────────┘    └────────────┘
     │               │                 │                   │
     ▼               ▼                 ▼                   ▼
  External       Check            Score changed        Watchlist
  (fear/FOMO)    K-Score          Insight found        Alerts
     +                             Rug avoided          Predictions
  Internal                         Gem discovered
  (daily alert)
```

**Triggers:**
- External: Fear of rug, FOMO on new token, CT discussion
- Internal: Daily summary notification, habit cue

**Variable Rewards:**
- K-Score went up → Confidence
- K-Score went down → Early warning (saved money!)
- New insight discovered → Alpha
- Prediction correct → Validation

**Habit Loop:**
```
Morning: Check portfolio K-Score summary
Before trade: Check token K-Score
After CT mention: Check token K-Score
Weekly: Review predictions, adjust watchlist
```

**Metric:** DAU/MAU ratio > 0.3 (30% daily actives).

## Metrics

### A-Score (Attention Health)

```
A-Score = 100 × ∜(I × E × C × H)

Where:
├── I = Impressions (reach)           Weight: 1.0
├── E = Engagement (clicks/views)     Weight: φ (1.618)
├── C = Conversion (actions/clicks)   Weight: φ² (2.618)
└── H = Habit (return rate)           Weight: φ³ (4.236)

INTERPRETATION:
├── A > 80: Excellent (viral growth)
├── A > 60: Good (sustainable growth)
├── A > 40: Fair (needs optimization)
└── A < 40: Poor (funnel broken)
```

### Funnel Metrics

| Metric | Definition | Target |
|--------|------------|--------|
| **CAC** | Cost per acquired user | < $1 |
| **Time to Value** | Seconds to first K-Score | < 30s |
| **Exploration Rate** | Users who check > 3 tokens | > 50% |
| **Investment Rate** | Users who create watchlist | > 20% |
| **DAU/MAU** | Daily/Monthly active ratio | > 30% |
| **Viral Coefficient** | New users per existing | > 1.0 |
| **LTV** | Lifetime value (burns) | > $10 |
